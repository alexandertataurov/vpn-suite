## Why it exists

Foundations define the rules and building blocks for consistent UI. Tokens (color, typography, spacing, etc.) ensure that components look and behave the same across the app. Using semantic tokens instead of raw values reduces drift and makes theming and accessibility easier.

## Core principles

- **Intro → tokens → visual primitives → layout primitives → accessibility guardrails**
- **Foundations** = rules and primitives
- **Components** = reusable UI units
- **Patterns** = multi-component solutions
- **Templates** = page-level composition

## Foundation map

| Foundation | Purpose |
|------------|---------|
| Color | Semantic color system for meaning, contrast, and interaction |
| Typography | Type scale, readability rules, and semantic text use |
| Spacing | 8px grid, insets, gaps, and layout rhythm |
| Layout | Breakpoints, grid, containers, and responsive rules |
| Accessibility | Contrast, focus, keyboard, touch targets, motion safety |
