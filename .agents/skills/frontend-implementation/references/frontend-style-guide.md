# Frontend style guide

## Visual priorities
- clarity over novelty
- consistency over variety
- hierarchy over decoration
- shipping quality over mockup theatrics

## Spacing
- follow the repo spacing scale
- prefer consistent vertical rhythm
- avoid one-off pixel values unless already common in the repo

## Typography
- reuse existing text styles
- keep heading levels meaningful
- do not invent extra display sizes for ordinary screens

## Color
- use existing tokens/classes only
- keep accent usage sparse
- error/success/warning styles should follow established patterns

## Components
- prefer existing UI primitives
- do not create alternate versions of the same component without strong need

## Interaction
- primary action must be obvious
- dangerous actions must look dangerous
- disabled state must remain legible
- focus state must be visible
