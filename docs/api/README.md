# VPN Suite API — Canonical Reference

This document is the **single source of truth** for the API contract. It is derived from code and runtime configuration.

- **Canonical OpenAPI:** `openapi/openapi.yaml` (generated by `./manage.sh openapi`)
- **Audit findings:** `docs/api/inconsistencies.md`

---

## Architecture Overview

**Control-plane (Admin API)**
- Service: `admin-api` (FastAPI)
- Paths: `/api/v1/*` plus non-versioned `/metrics`, `/health`, `/health/ready`, `/webhooks/*`

**Bot (Sales Gateway)**
- Service: `telegram-vpn-bot`
- Uses Admin API with `X-API-Key`

**Node Agent (Optional)**
- Service: `node-agent`
- Pulls desired state and reports telemetry
- HTTP health/metrics locally; control-plane traffic is HTTPS (mTLS)

**Data-plane (AmneziaWG)**
- UDP WireGuard/AmneziaWG only; no HTTP management API

---

## Base URLs

- **Public API (via reverse proxy):** `https://$PUBLIC_DOMAIN`
- **Local dev:** `http://localhost:8000`
- **Agent API (mTLS):** `https://$PUBLIC_DOMAIN:8443/api/v1/agent/*`

---

## Authentication Models

### Admin JWT (Bearer)
- Used by admin UI and operator tools.
- Obtain tokens via `/api/v1/auth/login` and `/api/v1/auth/refresh`.

### Bot API Key (X-API-Key)
- Used by Telegram bot to call `/api/v1/bot/*` and some mixed endpoints.

### WebApp Session (Bearer)
- WebApp obtains a short-lived session token via `/api/v1/webapp/auth` using Telegram `initData`.

### Agent API (mTLS + X-Agent-Token)
- mTLS enforced at reverse proxy for `/api/v1/agent/*`.
- Shared token enforced by API via `X-Agent-Token`.
 - Agent API returns 503 if `AGENT_SHARED_TOKEN` is not configured.

### Webhooks
- `/webhooks/payments/{provider}` is public but supports provider secret headers (Telegram Stars).

## Versioning & Compatibility

- **Primary versioning:** `/api/v1/*`
- **Compatibility alias (deprecated):** `/api/telemetry/docker/*` (alias for `/api/v1/telemetry/docker/*`)
- **Non-versioned:** `/metrics`, `/health`, `/health/ready`, `/webhooks/*`

## Error Handling Standard

All error responses follow this envelope:

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Validation error",
    "details": {
      "errors": []
    }
  },
  "meta": {
    "timestamp": "2026-02-24T05:00:00Z",
    "code": 422,
    "request_id": "..."
  }
}
```

---

## Pagination Standard

Most list endpoints use **limit/offset**:

```
GET /api/v1/users?limit=50&offset=0
```

`GET /api/v1/servers` also supports `page` + `page_size` (legacy override). This will be standardized (see `docs/api/inconsistencies.md`).

---

## Rate Limiting

Global rate limiting is enabled across the Admin API via middleware. Login failures also have specific rate limiting.

## Examples

### Admin Login

```bash
curl -s -X POST https://$PUBLIC_DOMAIN/api/v1/auth/login \
  -H 'Content-Type: application/json' \
  -d '{"email":"admin@example.com","password":"•••"}'
```

### List Servers (Admin)

```bash
curl -s https://$PUBLIC_DOMAIN/api/v1/servers?limit=20&offset=0 \
  -H 'Authorization: Bearer $ADMIN_JWT'
```

### Bot Create Invoice

```bash
curl -s -X POST https://$PUBLIC_DOMAIN/api/v1/bot/payments/telegram_stars/create-invoice \
  -H 'X-API-Key: $BOT_API_KEY' \
  -H 'Content-Type: application/json' \
  -d '{"plan_id":"plan_basic","tg_id":123456789}'
```

### Agent Heartbeat

```bash
curl -s -X POST https://$PUBLIC_DOMAIN:8443/api/v1/agent/heartbeat \
  -H 'X-Agent-Token: $AGENT_SHARED_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{"server_id":"vpn-node-1","container_name":"amnezia-awg","interface_name":"awg0","public_key":"...","listen_port":45790,"peer_count":0,"total_rx_bytes":0,"total_tx_bytes":0,"health_score":1.0,"status":"healthy","agent_version":"1.0.0","ts_utc":"2026-02-24T05:00:00Z"}'
```
