# Конфиг не подключается (нет handshake)

## Что проверить

### 1. Ключ сервера в конфиге совпадает с нодой

В выданном конфиге в секции `[Peer]` указан **PublicKey** — это публичный ключ интерфейса на VPN-ноде.

На сервере (хост с контейнером AmneziaWG) выполните:

```bash
docker exec <контейнер_awg> awg show awg0 public-key
# или
docker exec <контейнер_awg> wg show awg0 public-key
```

Строка должна **совпадать** с `PublicKey` из конфига. Если отличается — в конфиге устаревший ключ (например, после пересоздания контейнера).

**Что сделать:** в админке **Servers** → выберите сервер → **Sync**. Дождитесь выполнения (в agent mode агент подхватит задачу и обновит ключ в heartbeat). Затем один раз **Reissue** для девайса и скачайте новый конфиг.

### 2. Пир есть на ноде

Клиентский публичный ключ (он соответствует `PrivateKey` из конфига) должен быть добавлен на ноду как peer.

На сервере:

```bash
docker exec <контейнер_awg> awg show awg0
```

В списке peers должен быть ключ этого клиента. В agent mode пиры применяет node-agent по desired-state; если агент не отработал или упал, пира может не быть.

**Что сделать:** убедиться, что node-agent запущен и ходит на control-plane. Подождать 30–60 с после reissue и проверить `awg show awg0` снова. При необходимости в админке нажать **Sync** для сервера, затем снова **Reissue**.

### 3. Endpoint и сеть

В конфиге указан **Endpoint** (хост:порт). Убедитесь, что:

- порт открыт на файрволе (UDP);
- с клиента этот хост и порт достижимы.

### 4. H1–H4 и обфускация

AmneziaWG требует совпадения параметров обфускации (H1–H4, Jc, Jmin, Jmax, S1, S2) у клиента и сервера. Если на ноде пересоздали контейнер или поменяли переменные окружения (AWG_H1 и т.д.), конфиг со старыми H может не подключаться.

**Что сделать:** после смены конфигурации ноды выполнить **Sync** в админке и затем **Reissue** для девайса, чтобы выдать конфиг с актуальными параметрами.

---

## Краткий сценарий «конфиг не работает после reissue»

1. **Servers** → выбрать сервер этого девайса → **Sync** (обновить ключ и состояние ноды).
2. Подождать 30–60 секунд (в agent mode — пока агент применит desired-state).
3. **Devices** → нужный девайс → **Reissue** один раз.
4. Скачать новый конфиг и подключиться.
5. На ноде проверить: `awg show awg0` — есть ли peer с нужным публичным ключом; `awg show awg0 public-key` — совпадает ли с `PublicKey` в конфиге.
