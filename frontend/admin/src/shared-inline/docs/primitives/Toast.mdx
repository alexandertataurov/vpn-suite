# Toast

## Purpose
Transient toast notifications via useToast.

**When to use:** Short-lived success/error/info feedback.

**When not to use:** Critical errors or required action (use InlineAlert/PageError).

## API (Props)

| Prop | Type |
| --- | --- |
| useToast() | `{ addToast: (message: string, variant?: "success" \\| "error" \\| "info", duration?: number) => void }` |
| ToastContainer | `React.FC<{ children: ReactNode }>` |

## Examples

```tsx
<ToastContainer />
```

## Design Tokens Used
- Typography: `--text-*`
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Radius: `--radius-*`

## Anti-patterns
- Inline styles or arbitrary pixel values.
- Using component for unrelated semantics.

## Accessibility
- Provide labels and aria attributes as needed.
- Ensure focus visibility and keyboard access.
