# Confirmation Flows

Modal confirmations for user actions.

## Components

- **ConfirmModal** — Generic confirm/cancel
- **ConfirmDanger** — Destructive actions (delete, revoke)

## Pattern

1. User triggers action (e.g. "Delete")
2. Modal opens with title, message
3. Cancel (ghost) or Confirm (primary/danger)
4. onConfirm runs; modal closes

## Usage

```tsx
<ConfirmDanger
  open={open}
  onClose={() => setOpen(false)}
  onConfirm={handleDelete}
  title="Delete server"
  message="This cannot be undone."
/>
```
