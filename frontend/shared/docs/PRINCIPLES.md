# Design System Principles

1. Token-first: use semantic tokens only in components.
2. Primitive-first: primitives are the only styling source for composites.
3. Variant minimalism: keep variants and sizes minimal.
4. Strict APIs: union types, no stringly-typed variants.
5. Accessibility-first: keyboard, focus, ARIA by default.
6. Composition over inheritance: compose primitives.
7. Zero duplication: one component per role.
8. Documentation-driven: every component has Storybook docs.
