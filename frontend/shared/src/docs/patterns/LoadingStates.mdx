# LoadingStates

## Purpose
Consistent loading indicators for async operations.

**When to use:** Pending fetches, background sync, and skeleton placeholders.

**When not to use:** Empty data states (use EmptyState) or errors (use ErrorState).

## API (Props)

Use `Spinner`, `Skeleton`, and `ProgressBar` depending on context.

## Examples

```tsx
<SkeletonList lines={3} />
<Spinner size="sm" />
<ProgressBar value={42} />
```

## Design Tokens Used
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Radius: `--radius-*`
- Typography: `--text-*`

## Anti-patterns
- Mixing multiple loading styles in one surface.
- Using spinners for static page loading.

## Accessibility
- Use `aria-busy` where appropriate.
- Ensure focus is not trapped on loading elements.
