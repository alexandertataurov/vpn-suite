# EmptyStates

## Purpose
Guidance for empty data views using EmptyState.

**When to use:** Lists, tables, or dashboards with no results.

**When not to use:** Loading states (use Skeleton/Spinner) or errors (use ErrorState).

## API (Props)

Use the `EmptyState` component with `title`, `description`, and optional `action`.

## Examples

```tsx
<EmptyState
  title="No servers"
  description="Add your first server to get started."
  action={<Button variant="primary">Add server</Button>}
/>
```

## Design Tokens Used
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Typography: `--text-*`

## Anti-patterns
- Showing empty states for transient loading.
- Replacing errors with empty states.

## Accessibility
- Ensure heading hierarchy is preserved.
- Actions must be keyboard accessible.
