# StatusBadge

## Purpose
Opinionated OK/Degraded/Down/Unknown badge.

**When to use:** Health/status signals for systems.

**When not to use:** Arbitrary categories (use Badge).

## API (Props)

| Prop | Type |
| --- | --- |
| status | `"ok"` | "degraded" | "down" | "unknown" |
| label? | `string` |
| nativeProps | `Omit<HTMLAttributes<HTMLSpanElement>, "children">` |

## Examples

```tsx
<StatusBadge status="degraded" />
```

## Design Tokens Used
- Typography: `--text-*`
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Radius: `--radius-*`

## Anti-patterns
- Inline styles or arbitrary pixel values.
- Using component for unrelated semantics.

## Accessibility
- Provide labels and aria attributes as needed.
- Ensure focus visibility and keyboard access.
