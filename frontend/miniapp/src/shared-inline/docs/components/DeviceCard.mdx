# DeviceCard

## Purpose
Device summary card with status badge and actions.

**When to use:** Device lists.

**When not to use:** Dense tabular data (use Table).

## API (Props)

| Prop | Type |
| --- | --- |
| id | `string` |
| name? | `string` |
| status | `"active"` | "revoked" |
| issuedAt | `string` |
| shortId? | `string` |
| primaryAction? | `ReactNode` |
| secondaryActions? | `ReactNode` |

## Examples

```tsx
<DeviceCard id="abc" status="active" issuedAt="..." />
```

## Design Tokens Used
- Typography: `--text-*`
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Radius: `--radius-*`

## Anti-patterns
- Inline styles or arbitrary pixel values.
- Using component for unrelated semantics.

## Accessibility
- Provide labels and aria attributes as needed.
- Ensure focus visibility and keyboard access.
