# PageError

## Purpose
Full-page error panel.

**When to use:** Fatal page failures.

**When not to use:** Inline errors (use ErrorState).

## API (Props)

| Prop | Type |
| --- | --- |
| title? | `string` |
| message? | `string` |
| requestId? | `string` |
| statusCode? | `number` |
| endpoint? | `string` |
| onRetry? | `() => void` |

## Examples

```tsx
<PageError title="Failed" onRetry={retry} />
```

## Design Tokens Used
- Typography: `--text-*`
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Radius: `--radius-*`

## Anti-patterns
- Inline styles or arbitrary pixel values.
- Using component for unrelated semantics.

## Accessibility
- Provide labels and aria attributes as needed.
- Ensure focus visibility and keyboard access.
