# ProfileCard

## Purpose
Subscription/profile summary card.

**When to use:** Account overview sections.

**When not to use:** Generic info panels (use Panel).

## API (Props)

| Prop | Type |
| --- | --- |
| planName? | `string` |
| planId? | `string` |
| validUntil | `string` |
| status | `string` |
| primaryAction? | `ReactNode` |

## Examples

```tsx
<ProfileCard planName="Pro" validUntil="..." status="active" />
```

## Design Tokens Used
- Typography: `--text-*`
- Spacing: `--spacing-*`
- Colors: `--color-*`
- Radius: `--radius-*`

## Anti-patterns
- Inline styles or arbitrary pixel values.
- Using component for unrelated semantics.

## Accessibility
- Provide labels and aria attributes as needed.
- Ensure focus visibility and keyboard access.
