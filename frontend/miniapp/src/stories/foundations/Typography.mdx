## Why it exists

Typography creates hierarchy and reading comfort. Semantic text styles ensure consistent sizing and weight across the app.

## Rules

- Use semantic text styles (`--typo-*`), not arbitrary font sizes
- Limit font weights (400, 500, 600, 700)
- Respect max line length for long-form content

## Implementation notes

- Tokens: `--typo-display-size`, `--typo-h1-size`, `--typo-body-size`, `--typo-caption-size`, etc.
- Font family: `var(--font-sans)` for UI, `var(--font-mono)` for code
- Responsive: consumer theme overrides in `theme/consumer.css`

## Accessibility

- Minimum body size: 14px
- Line height: 1.4 for body, 1.2 for headings
- Contrast: text on background must meet WCAG AA
