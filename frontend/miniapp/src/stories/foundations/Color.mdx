## Why it exists

Color communicates hierarchy, action, feedback, and state. Semantic tokens must be used instead of raw palette values so the system stays consistent across themes and surfaces.

## Rules

- Use semantic tokens in product code
- Do not use raw hex values in components
- Interactive colors require hover / active / disabled states
- Text/background pairs must pass contrast requirements

## Implementation notes

- Token aliasing: `--color-bg`, `--color-surface`, `--color-accent`, etc.
- Light/dark: `data-theme="consumer-light"` and `data-theme="consumer-dark"` in `theme/consumer.css`
- Use `var(--color-*)` in CSS; avoid hardcoded hex

## Accessibility

- Contrast: text/background pairs must meet WCAG AA (4.5:1 for body, 3:1 for large text)
- Non-color indicators: use icons or labels for status, not color alone
- Focus outline: `--color-focus-ring` for visible focus states
